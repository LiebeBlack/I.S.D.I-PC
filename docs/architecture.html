1

Infrastructure Layer

SQLite · File System · APIs Externas

Lo que toca el disco duro. Acá vive la conexión real con SQLite, la lectura de archivos del sistema y cualquier API externa. Si mañana cambiás SQLite por PostgreSQL, solo tocás esta capa.

Python
import sqlite3
from domain.ports import UserRepository

class SQLiteUserRepository(UserRepository):
    def __init__(self, db_path: str):
        self.conn = sqlite3.connect(db_path)

    def find_by_id(self, user_id: int):
        cursor = self.conn.execute(
            "SELECT * FROM users WHERE id = ?",
            (user_id,)
        )
        return cursor.fetchone()
2

Domain Layer

Entities · Value Objects · Interfaces

La parte más importante y la que menos cambia. Acá están las entidades (User, Course, etc.), las reglas de negocio y las interfaces que las capas de afuera tienen que respetar. Cero dependencias externas.

Python
from abc import ABC, abstractmethod
from dataclasses import dataclass

@dataclass
class User:
    id: int
    name: str
    email: str
    role: str = "student"

class UserRepository(ABC):
    @abstractmethod
    def find_by_id(self, user_id: int) -> User: ...
3

Application Layer

Use Cases · Controllers · DTOs

Los casos de uso. Cada acción que el usuario puede hacer ("ver mi perfil", "crear tarea") es una clase acá. Coordina entre el dominio y la infraestructura, pero no sabe qué base de datos estás usando.

Python
class GetUserUseCase:
    def __init__(self, repo: UserRepository):
        self.repo = repo

    def execute(self, user_id: int) -> User:
        user = self.repo.find_by_id(user_id)
        if not user:
            raise ValueError("Usuario no encontrado")
        return user
4

Presentation Layer

Flet · Flutter Engine · UI Components

Lo que ve el usuario. Pantallas hechas con Flet que llaman a los use cases de la capa de arriba y muestran los datos. Botones, formularios, listas, navegación — todo acá.

Python
import flet as ft

class UserPage(ft.UserControl):
    def __init__(self, use_case: GetUserUseCase):
        super().__init__()
        self.use_case = use_case

    def build(self):
        user = self.use_case.execute(1)
        return ft.Column([
            ft.Text(user.name, size=24),
            ft.Text(user.email, color="grey"),
        ])
Principios

Principios SOLID

S

Single Responsibility

Una clase, una responsabilidad. El repositorio guarda datos y punto. El use case orquesta lógica y punto.

O

Open/Closed

Querés agregar un nuevo tipo de reporte? Lo extendés, no modificás lo que ya funciona.

L

Liskov Substitution

Si cambiás SQLiteUserRepository por MongoUserRepository y la app sigue funcionando, estás cumpliendo Liskov.

I

Interface Segregation

Interfaces chicas y específicas. Cada interfaz define solo lo que su consumidor necesita.

D

Dependency Inversion

El dominio define una interfaz y la infraestructura la implementa. Las dependencias siempre apuntan hacia adentro.
